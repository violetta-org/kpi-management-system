# 🚀 Power Apps VIBE: Export, Deployment & GitHub Source Control Guide

This guide details the complete Application Lifecycle Management (ALM) workflow for applications built using the **Power Apps VIBE Experience** (`vibe.powerapps.com`). Since Vibe generates Dataverse tables, custom React code, and Canvas structures, managing them through solutions, pipelines, and GitHub source control is essential for enterprise deployments.

## ⚙️ Prerequisites

Before you begin, ensure you have the following tools installed and configured:

1. **Power Platform CLI (`pac`):** The primary command-line interface tool for Power Platform.
   - Install via [Microsoft Learn: Install Power Platform CLI](https://learn.microsoft.com/power-platform/developer/cli/introduction).
2. **Git:** For source control tracking.
   - Install via [git-scm.com](https://git-scm.com/).
3. **GitHub Account & Repository:** To host your codebase.
4. **Conda Environment (Python):** For running the automation script.
   - Ensure Python 3.8+ is installed in your Conda environment.

## 💡 Vibe Development Best Practice

> [!IMPORTANT]
> **Preferred Solutions Context:**
> Vibe applications are stored as components in your Power Platform environment. To avoid your app being scattered or locked in the default solution, you **must set a Preferred Solution** before starting your build:
> 1. Go to [make.powerapps.com](https://make.powerapps.com).
> 2. Click **Solutions** in the left navigation.
> 3. Click the **Preferred Solution** dropdown in the top-right toolbar and select (or create) the solution you wish to contain your Vibe project.
> 4. Go to [vibe.powerapps.com](https://vibe.powerapps.com) and build your application. All generated components (tables, React components, and apps) will automatically be saved into that preferred solution.

## 📤 Step 1: Exporting the Solution

To move your app out of the development environment, you must export the solution containing it.

### Option A: Manual Export via Maker Portal
1. Go to [make.powerapps.com](https://make.powerapps.com) and navigate to **Solutions**.
2. Select your solution and click **Export** from the top menu.
3. Select **Publish all changes** to ensure the latest updates are compiled.
4. Set the version number and export type:
   - **Managed:** Use this for deploying to **Test** or **Production** environments (prevents editing in the target environment).
   - **Unmanaged:** Use this for checking into **GitHub source control** or deploying to another developer environment (allows edits).
5. Click **Export** and download the `.zip` file.

### Option B: CLI Export
Authenticate and export using Power Platform CLI:
```bash
# 1. Authenticate to your developer environment
pac auth create --name DevEnv --url https://<your-dev-org>.crm.dynamics.com

# 2. Export the solution as Unmanaged (for source control)
pac solution export --name "YourSolutionName" --zipfile "YourSolutionName.zip" --managed false
```

## 🐙 Step 2: GitHub Source Control Integration

Once you have your solution zip file, you need to unpack it into human-readable source code and commit it to GitHub.

### 1. Initialize your local Git Repository
In your local workspace folder (e.g., `vibepowerapps`):
```bash
git init
```

### 2. Unpack the Solution
Use the PAC CLI to extract the compressed solution zip into human-readable XML, JSON, and React source files:
```bash
pac solution unpack --zipfile "YourSolutionName.zip" --folder "./src"
```
This breaks down the solution package into:
*   `CanvasApps/`: The `.msapp` files and source representations.
*   `Entities/`: Dataverse table metadata definitions.
*   `WebResources/`: Custom React or TypeScript component code generated by the Vibe engine.

### 3. Create a `.gitignore`
Create a `.gitignore` file to avoid tracking temporary solution archives:
```text
*.zip
*.log
.bin/
obj/
bin/
```

### 4. Commit and Push to GitHub
```bash
git add .
git commit -m "feat: initial commit of vibe-generated application"
git branch -M main
git remote add origin https://github.com/your-username/your-repo-name.git
git push -u origin main
```

## 🚀 Step 3: Deploying to Target Environments

There are three primary ways to deploy your application:

### Method A: Manual Import
1. Go to [make.powerapps.com](https://make.powerapps.com) in your target environment (e.g., Production).
2. Go to **Solutions** -> **Import**.
3. Select the **Managed** solution `.zip` file you exported in Step 1.
4. Click **Next**, configure any environment variables or connection references, and click **Import**.

### Method B: Native Power Platform Pipelines
1. Set up **Pipelines** in your tenant (Power Platform Admin Center).
2. From the developer portal in `make.powerapps.com`, click **Deploy** on your solution.
3. Select your target stage (e.g., Test, Production) and run the pipeline. Power Platform handles the export, conversion to managed, and deployment automatically.

### Method C: GitHub Actions CI/CD (Automated ALM)
You can automate the build and deployment process with a GitHub Actions workflow using **Microsoft Power Platform Build Tools**.
Create a workflow file `.github/workflows/deploy.yml`:
```yaml
name: Deploy Solution

on:
  push:
    branches:
      - main

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v3

    # Pack the source files back into a solution zip
    - name: Pack Solution
      uses: microsoft/powerplatform-actions/pack-solution@v0
      with:
        solution-folder: 'src'
        solution-file: 'out/YourSolutionName.zip'
        solution-type: 'Managed'

    # Import the managed solution to the production environment
    - name: Import Solution to Production
      uses: microsoft/powerplatform-actions/import-solution@v0
      with:
        environment-url: 'https://<your-prod-org>.crm.dynamics.com'
        app-id: ${{ secrets.CLIENT_ID }}
        client-secret: ${{ secrets.CLIENT_SECRET }}
        tenant-id: ${{ secrets.TENANT_ID }}
        solution-file: 'out/YourSolutionName.zip'
        force-overwrite: true
        publish-changes: true
```

## 🐍 Python Automation Script (Conda Compatible)

To automate the export, unpack, and Git commit workflow, we have provided an `automate_alm.py` script. 

### How to use with Conda:
1. Open your Conda prompt and activate your Python environment:
   ```bash
   conda activate your_env_name
   ```
2. Set your environment variables or edit the configuration block in `automate_alm.py`.
3. Run the script to export, unpack, and commit in one go:
   ```bash
   python automate_alm.py --export --unpack --commit
   ```
